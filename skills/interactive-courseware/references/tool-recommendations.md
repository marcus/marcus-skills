# Tool Recommendations Reference

Evaluated tools for agent-driven courseware creation, ranked by fitness for the pipeline.

## Tier 1: Recommended Core Stack

### scorm-again — SCORM Runtime

- **Repo:** github.com/jcputney/scorm-again
- **License:** MIT
- **What:** Modern SCORM 1.2 and SCORM 2004 JavaScript runtime
- **Why recommended:**
  - TypeScript-first, well-tested
  - Cross-Frame Communication (v3.0.0+) for sandboxed iframes
  - Works without backing LMS (development/testing mode)
  - Separate SCORM 1.2 and SCORM 2004 implementations
  - Active maintenance
- **Agent relevance:** Library, not a content generator — used as a dependency

### Vite — Build System

- **URL:** vitejs.dev
- **License:** MIT
- **Why recommended:**
  - Fast HMR for rapid content iteration
  - Native ES modules in development
  - Optimized production builds with Rollup
  - Code splitting for lazy-loaded course sections
  - Plugin ecosystem for custom build steps
- **Agent relevance:** CLI-driven build, easily automated

### GSAP + ScrollTrigger — Animation

- **URL:** gsap.com
- **License:** Free for most uses (see GSAP license)
- **Why recommended:**
  - Industry-standard animation performance
  - ScrollTrigger for scroll-driven storytelling
  - SVG morphing for diagram animations
  - Timeline management for complex sequences
- **Agent relevance:** Code-driven, animations can be generated from content descriptions

### React Three Fiber — 3D Content (When Needed)

- **Repo:** github.com/pmndrs/react-three-fiber
- **License:** MIT
- **Why recommended:** React component model for Three.js — familiar patterns, good performance
- **When to use:** Only when 3D adds genuine understanding over 2D
- **Agent relevance:** React components can be templated

### Sandpack — Code Playgrounds

- **Repo:** github.com/codesandbox/sandpack
- **License:** Apache 2.0
- **Why recommended:**
  - React components for live-running code editors
  - Supports many frameworks (Next.js, Vite, Remix, Astro)
  - Nodebox for server-side code in browser
  - Themeable, with file explorer and debug tools
- **Agent relevance:** Code examples can be injected as props

### Mermaid.js — Diagrams

- **Repo:** github.com/mermaid-js/mermaid
- **License:** MIT
- **Why recommended:** Text-based diagrams, renders in browser, LLM-friendly syntax
- **Agent relevance:** Excellent — Mermaid syntax is trivially generated by LLMs
- **Skill reference:** See `technical-diagrams` skill

## Tier 2: Valuable for Specific Use Cases

### Twine / Twee — Branching Scenarios

- **URL:** twinery.org
- **License:** GPL
- **Why valuable:**
  - Purpose-built for interactive nonlinear stories
  - Twee text format is one of the simplest for LLM generation
  - Popular in L&D for scenario-based training
  - Variables, conditional logic, CSS, JavaScript support
- **Limitations:** No native SCORM support (needs wrapper), no built-in assessment
- **Agent relevance:** Excellent — Twee is trivially LLM-generated
- **Best for:** Standalone branching scenarios, rapid prototyping

### Adapt Framework — Full SCORM Courseware

- **URL:** adaptlearning.org
- **Repo:** github.com/adaptlearning/adapt_framework (618 stars)
- **License:** GPL v3
- **Why valuable:**
  - Most mature open-source SCORM authoring solution
  - Responsive HTML5 output, WCAG 2.1 AA accessible
  - 23 bundled plugins, extensive community ecosystem
  - JSON configuration files define course structure
- **Limitations:** Steep learning curve, authoring tool requires MongoDB
- **Agent relevance:** Good — generate JSON config files + CLI build
- **Best for:** When you need a proven SCORM player without building your own

### H5P — Interactive Content Types

- **URL:** h5p.org
- **License:** GPL v3
- **Why valuable:**
  - 50+ interactive content types (Interactive Video, Branching Scenario, Quiz types, etc.)
  - Integrates with WordPress, Moodle, Drupal
  - Built-in xAPI statement emission
  - Largest ecosystem of interaction types in open source
- **Limitations:** SCORM export not native, requires host platform
- **Agent relevance:** Good via EscolaLMS headless API (REST CRUD for H5P content)
- **Best for:** Embedding interactions in existing CMS/LMS platforms

### Reveal.js — Presentation-Based Courses

- **URL:** revealjs.com
- **Repo:** github.com/hakimel/reveal.js (70,700 stars)
- **License:** MIT
- **Why valuable:**
  - Markdown content support, plugin architecture
  - Auto-Animate transitions, speaker notes, PDF export
  - JavaScript API for extending with quizzes, tracking
  - Extremely mature (787K dependents)
- **Agent relevance:** Excellent — Markdown or HTML trivially generated
- **Best for:** Quick course prototypes, presentation-heavy content, conference workshops

### Slidev — Developer-Focused Presentations

- **URL:** sli.dev
- **License:** MIT
- **Why valuable:**
  - Markdown + Vue 3 components for rich interactivity
  - Built-in Monaco Editor for live coding demos
  - Mermaid diagrams, LaTeX math, recording capability
  - Export to PDF, PNG, PPTX, or SPA
- **Agent relevance:** Excellent — Markdown + Vue components easily generated
- **Best for:** Programming courses, developer training, tech workshops

### Marimo — Interactive Python Notebooks

- **URL:** marimo.io
- **License:** Apache 2.0
- **Why valuable:**
  - Reactive Python notebook (cells auto-update on dependency change)
  - Stored as pure Python files (Git-friendly, LLM-friendly)
  - Interactive UI elements bound to Python without callbacks
  - Deploy as web apps, run in browser via WASM
  - Dedicated educational initiatives
- **Agent relevance:** Excellent — pure Python files are ideal for LLM generation
- **Best for:** Data science courses, Python training, interactive tutorials

### Phaser — Game-Based Learning

- **URL:** phaser.io
- **License:** MIT
- **Why valuable:** Premier 2D HTML5 game framework, physics, sprites, effects
- **Agent relevance:** Good — scene-based JS architecture is modular and templatable
- **Best for:** Gamified learning modules, educational games, simulation exercises

### OATutor — Adaptive Tutoring

- **URL:** oatutor.io
- **License:** MIT (code), CC BY 4.0 (content)
- **Why valuable:**
  - Bayesian Knowledge Tracing for skill mastery estimation
  - Built-in A/B testing for learning experiments
  - GPT integration variant for AI tutoring
  - Open source from UC Berkeley
- **Agent relevance:** Good — JSON-structured problem content
- **Best for:** STEM adaptive tutoring, math courses

### Excalidraw — Collaborative Whiteboard

- **URL:** excalidraw.com
- **License:** MIT (119K stars)
- **Why valuable:**
  - Embeddable React component, real-time collaboration
  - Hand-drawn aesthetic reduces intimidation
  - Frames feature for creating presentations
- **Agent relevance:** Moderate — can embed for collaborative exercises
- **Best for:** Collaborative activities, brainstorming exercises, visual thinking

## Tier 3: Specialized / Emerging

### JupyterLite — Browser Jupyter

- Zero-install Jupyter via WASM, static deployment, standard notebook format
- Best for: Data science courses where full Jupyter is needed

### Observable Framework — Data-Driven Content

- Markdown + reactive JavaScript, built-in D3/Plot/Mermaid
- Best for: Data journalism courses, interactive data exploration

### Quarto — Scientific Publishing

- Multi-language (R, Python, Julia), Reveal.js output, academic formatting
- Best for: Academic/research courses, data science curricula

### GDevelop — No-Code Games

- Visual event system, cross-platform export, JSON project format
- Best for: When non-programmers need to create educational games

## Decision Matrix

### Choosing a Course Architecture

```
Start
  │
  ├─ Need SCORM for corporate LMS?
  │   ├─ Yes ──▶ Need rich interactivity?
  │   │           ├─ Yes ──▶ Custom SPA + scorm-again
  │   │           └─ No ──▶ Adapt Framework
  │   └─ No ──▶ Standalone web deployment?
  │               ├─ Yes ──▶ What content type?
  │               │           ├─ Presentations ──▶ Reveal.js or Slidev
  │               │           ├─ Interactive notebooks ──▶ Marimo or JupyterLite
  │               │           ├─ Data exploration ──▶ Observable Framework
  │               │           └─ Full course ──▶ Custom SPA
  │               └─ No ──▶ What platform?
  │                           ├─ Moodle/WordPress ──▶ H5P
  │                           ├─ Open edX ──▶ Open edX Studio + AI Creator
  │                           └─ Custom LMS ──▶ xAPI/cmi5 + Custom SPA
  │
  └─ Just branching scenarios?
      └─ Yes ──▶ Twine/Twee (export to HTML, wrap in SCORM if needed)
```

### Choosing Multimedia Tools

```
Narration needed?
  └─ Yes ──▶ ElevenLabs TTS (eleven_multilingual_v2)

Video presenter needed?
  ├─ Yes, enterprise ──▶ Synthesia
  ├─ Yes, flexible ──▶ HeyGen
  └─ No, programmatic video ──▶ Remotion

Images needed?
  ├─ Diagrams ──▶ Mermaid.js
  ├─ Illustrations with labels ──▶ OpenAI GPT Image
  ├─ Consistent brand style ──▶ Stable Diffusion + LoRA
  └─ Quick concept art ──▶ DALL-E / Midjourney

Sound design needed?
  ├─ Background music ──▶ ElevenLabs Music
  ├─ UI sound effects ──▶ ElevenLabs SFX
  └─ Custom audio ──▶ Web Audio API (programmatic)

Interactive practice needed?
  ├─ Code exercises ──▶ Sandpack
  ├─ Voice role-play ──▶ ElevenLabs Agents
  ├─ Data exploration ──▶ Observable / D3.js
  └─ 3D models ──▶ React Three Fiber
```
