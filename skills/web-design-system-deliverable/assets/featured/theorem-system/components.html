Component reference

Low chrome by default. Filled containers are the exception.

Components default to text-only, dots, and bare affordances. Filled containers are reserved for the single highest-priority instance. Every variant shown here is production-ready.

Buttons

Shape and chrome hierarchy, not just color. Ghost and text buttons are the most common; filled primary is the exception. Six tiers from maximum to minimum visual weight.

Full priority ladder

Primary (1 per page): pill, accent fill, weight 450. Secondary: 6px radius, 1px border. Swiss: no border/fill, scale+shadow hover. Ghost: text+arrow, arrow slides on hover. Editorial: serif italic, underline-from-center hover.

Ghost / text+arrow buttons

No container. Text + arrow glyph. Arrow gap widens 4px on hover. This is the primary navigation affordance at Apple, Notion, and Vercel. Should be the most common button type.

Swiss minimal buttons

No border, no fill at rest. Hover lifts via scale(1.04) + shadow-sm. Active presses to scale(0.98). Works for secondary actions in any context.

Editorial buttons

Serif italic, no container. 1px underline grows from center on hover. Best for content-heavy, editorial contexts.

Luxury / fashion buttons

Black fill, ALL-CAPS, 0.2em letter-spacing, sharp corners. Inverts in dark mode. For high-fashion or editorial brand directions.

Danger + icon buttons

Danger: 6px radius, danger fill. Only for destructive actions. Icon: 36px square, 6px radius, 1px border.

States — hover

Primary/secondary: scale(1.015). Swiss: scale(1.04) + shadow-sm. 160ms with cubic-bezier(0.22, 0.61, 0.36, 1).

States — focus visible

2px accent ring, 2px offset. Uses :focus-visible so mouse users don't see focus rings.

States — disabled

Opacity 0.4, pointer-events: none, cursor: not-allowed.

States — loading

Text hidden, centered spinner. Pointer events disabled during loading.
