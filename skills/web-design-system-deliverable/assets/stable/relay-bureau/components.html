Component inventory

Action hierarchy first. Decoration second.

Relay Bureau uses shape, density, and tone to differentiate intent. Primary actions are pills. Secondary actions are rounded rectangles. Utility controls compress. Structural UI relies on rails and panels, not endless card stacks.

Coverage
  • Buttons, chips, navigation, fields, tables, alerts, empty states, drawers, modal framing, skeletons.
  • State coverage shown: default, hover-equivalent emphasis, focus treatment, loading, disabled, error, success, empty, skeleton.
  • Accessibility notes assume keyboard-first use, clear labelling, and visible live-region feedback on async states.
Buttons and chips

Shape carries action level.

The system avoids making every control a pill. Primary actions get the only full-pill treatment. Secondary and ghost controls stay structured; utility controls compress into compact rounded squares.

Navigation and command framing

Navigation should read like a production board, not a landing page header.

Instrument’s chip navigation and Koto’s modular work surfaces were useful references here. The product shell uses horizontal command bars and vertical rail navigation rather than top-heavy card menus.

Fields and form states

Inputs stay quiet until they need to speak.

Default inputs sit on translucent light surfaces. Validation state uses border and helper copy together; color alone is never asked to carry meaning. Focus is visible but not fluorescent.

Tables and lists

Data surfaces run compact. Summary surfaces breathe.

This is one of the main places the system diverges from portfolio theatrics. Dense work UI should feel almost journal-like: thin rules, tabular metadata, tight rows, and action affordances that don’t shout.

Feedback patterns

Alerts are flat, tinted, and direct. No thick left borders. Ever.

The skill explicitly calls out the “fat left strip on a rounded card” as an AI tell. Relay Bureau uses even borders, iconless tint, and compact copy blocks instead.

Overlays

Drawers and modals behave like review surfaces, not marketing interruptions.

Use drawers for inspect-and-return flows. Use modals only when the decision truly blocks the next branch. Both patterns keep their typography compact and their actions blunt.
