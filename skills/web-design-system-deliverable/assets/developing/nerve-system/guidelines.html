Guidelines

Design Guidelines

The rules, patterns, and conventions that keep Nerve consistent across every surface. This is the reference guide for engineers and designers working within the system.

Design Principles

Four tenets govern every design decision in Nerve. They are ranked; when principles conflict, the higher-ranked principle wins.

Principle 01
Decisive over decorative

Every element earns its place through utility. Ornamentation is stripped unless it improves comprehension or reinforces hierarchy. If a visual is purely decorative, remove it.

Principle 02
Density is a feature

Operators monitor many agents at once. Embrace information density. Whitespace serves structure, not decoration. Let users see more, scroll less.

Principle 03
Alive, not animated

Motion conveys state, not personality. Pulses mean live data. Transitions confirm actions. Nothing moves without reason. If you remove the animation and lose information, keep it; otherwise, cut it.

Principle 04
Dark is default

Operators work in low-light, extended sessions. The dark theme is the canonical design surface. Light mode is a supported alternative, never the starting point.

Do
Use a single accent color to draw attention to the primary action on screen. Let data density speak for itself with clean alignment and subtle separators.
Don't
Add gradients, drop shadows, or decorative illustrations to "make it feel premium." Nerve's premium quality comes from precision and clarity, not embellishment.
Do
Use a pulsing indicator to show that a data stream is live. Use a brief scale transition when a card is selected to confirm the interaction.
Don't
Add bouncing, spinning, or looping animations to entertain users. Motion without informational value is noise that fatigues operators during long sessions.

Measure & Line Length

Ideal line length is 55-75 characters for body text. Use content containers to enforce measure. The --content-narrow (42rem) container targets ~65 characters at body size.

/* Enforce readable measure */
.prose {
  max-width: var(--content-narrow);  /* ~65ch at --text-1 */
  line-height: var(--leading-body);
}

/* Wide measure for dashboards */
.dashboard-text {
  max-width: var(--content-medium);  /* acceptable for short-form UI copy */
}
Do
Use Space Grotesk for headings and Inter for body. Use tabular lining figures (lining-nums tabular-nums) in data tables so columns align vertically.
Don't
Use the display font (Space Grotesk) for body text or long paragraphs. Its geometric shapes cause reading fatigue at body sizes. Never use more than 3 heading levels on a single page.

Choreography Patterns

Use stagger delays to sequence related elements entering the screen. The base stagger delay is --motion-stagger-delay (50ms).

  animation: stagger-enter var(--motion-duration-slow) var(--motion-ease-emphasis) both;
}
.agent-list-item:nth-child(1) { animation-delay: calc(var(--motion-stagger-delay) * 0); }
.agent-list-item:nth-child(2) { animation-delay: calc(var(--motion-stagger-delay) * 1); }
.agent-list-item:nth-child(3) { animation-delay: calc(var(--motion-stagger-delay) * 2); }
.agent-list-item:nth-child(4) { animation-delay: calc(var(--motion-stagger-delay) * 3); }

/* Tab switch — crossfade panels */

/* Cascade reveal — dashboard widgets */

Reduced Motion

Reduced motion is a parallel design track, not an afterthought. When prefers-reduced-motion: reduce is active, all duration tokens resolve to 0ms. Design reduced-motion states that convey the same information through instant state changes rather than transitions.

Never disable animation entirely for reduced-motion users. Replace it. A shimmer skeleton becomes a static skeleton with a visible loading label. A stagger entrance becomes an instant render. The information must still be conveyed.

/* Reduced motion: replace shimmer with static + label */
@media (prefers-reduced-motion: reduce) {
  .skeleton {
    animation: none;
    background: var(--skeleton-base);
  }
  .skeleton::after {
    content: 'Loading...';
    font-size: var(--text-00);
    color: var(--color-fg-subtle);
  }
}
Do
Use --motion-duration-fast for hover states. Use stagger delays to reveal dashboard widgets sequentially so users can track the content appearing.
Don't
Use --motion-duration-dramatic for routine interactions. Long durations on common actions (button clicks, tab switches) feel sluggish and block the user.

Voice & Tone Guide

Nerve speaks with precision and restraint. The voice is consistent; the tone adapts to context. In an error, be direct and helpful. In onboarding, be encouraging without being casual.

Brand Vocabulary

Agent
An autonomous unit that executes tasks within defined guardrails. Never "bot" or "AI assistant."
Workflow
A sequence of tasks orchestrated across one or more agents. Never "pipeline" or "flow."
Deploy
To activate an agent. Never "launch" or "start." Deploy implies configuration is complete.
Guardrail
Execution boundary that constrains agent behavior. Never "limit" or "restriction."
Fleet
A collection of agents managed together. Never "pool" or "group."
Orchestrate
To coordinate agent execution. The core verb of the platform. Never "manage" or "run."
Escalation
When an agent transfers control to a human operator. Never "handoff" or "fallback" (fallback is for automated retry).
Nerve
The platform. Always capitalized. Never "NERVE" or "nerve" in copy. Acceptable as a code reference in lowercase.

Color Independence

Never rely on color alone to convey information. Every color-coded element must also use an icon, label, pattern, or position to communicate its meaning.

Do
Show status as a colored badge with an icon and label: a green checkmark with "Healthy" text. The icon + text convey meaning even without color perception.
Don't
Use a green or red dot as the only indicator of agent health. Color-blind users cannot distinguish red from green dots without an accompanying label.

Versioning Strategy

The token package follows semantic versioning (semver). Components are versioned independently but reference a compatible token range.

/* Token package versioning */
@nerve/tokens@2.4.1
  ^major: breaking token changes (renames, removals)
  ^minor: new tokens added
  ^patch: value adjustments (non-breaking)

/* Component package references */
@nerve/components@3.1.0
  peerDependency: "@nerve/tokens": "^2.0.0"

Deprecation Protocol

Deprecation is a 3-step process spread across at least 2 minor versions.

StepVersionAction
1. Announcev2.5.0Mark as deprecated in docs. Add /** @deprecated */ JSDoc. Emit console warning in dev mode.
2. Aliasv2.6.0Old name aliases to new name. Migration codemod published. Warning persists.
3. Removev3.0.0Old name removed. Major version bump. Breaking change logged in CHANGELOG.
